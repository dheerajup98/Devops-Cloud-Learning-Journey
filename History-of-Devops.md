# üîπ What is DevOps?

**DevOps** is a culture, practice, and automation-driven approach that integrates **Development (Dev)** and **Operations (Ops)** teams to improve collaboration, speed, reliability, and software delivery efficiency.

DevOps focuses on:

- Continuous Integration (CI)
- Continuous Delivery/Deployment (CD)
- Infrastructure as Code (IaC)
- Automation
- Monitoring & Observability
- Collaboration & Shared Responsibility

DevOps aims to deliver software:

- Faster
- More reliably
- With fewer failures
- With automated workflows

## üîπ Before DevOps (Pre-2008 Era)

Around 10‚Äì15 years ago, software development and IT operations worked in completely separate silos.

### Development Team:
- Wrote application code
- Focused on features
- Goal: Deliver new functionality

### Operations / System Admin Team:
- Managed servers
- Deployed applications manually
- Maintained uptime and infrastructure
- Goal: Keep production stable

### Build & Release Engineers:
- Managed build processes
- Created deployment packages
- Coordinated between dev and ops
- Often manually executed deployment steps

# üî• The Pain of Build & Release Engineers (Before DevOps)

Build & Release Engineers faced major challenges:

- Manual build creation
- Dependency conflicts
- Deployment script failures
- Late-night production releases
- No automated rollback strategy
- High pressure during production outages

They acted as the bridge but lacked automation tools like:

- Jenkins
- GitHub Actions
- Docker
- Kubernetes
- Terraform
